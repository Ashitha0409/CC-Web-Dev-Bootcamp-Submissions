##
Users and oppurtunites they post are linked by id in oppurtunity.json

In applications.json, the user id and the oppurtunity they have applied for are linked
As students might to apply to many oppurtunities, storing all the events/internships they have applied for in the user document might make it too large. Hence, easier to store the id of the intership and the applicants who have applied for it

## Smart Feature- 
In each oppurtunity there is a required skills field. The system can check the skills of the users and give an eligibility score.
There is also a deadline field. An algorithm can push the oppurtunities which are nearing the deadline upwards.
